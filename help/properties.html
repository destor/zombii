Property Index

zombie.tf

/aide string
The name of the current party aide (if any). This setting is automatically configured when typing "party status".

/allow_pings boolean
Allow others to trigger your lag check via the "ping" command.

/announce boolean
Should announcements be sent? This is the overall setting that controls announcements. When this is off all macros which use "/do_announce" will be disabled.

/announce_other_command string
The command that is used when /announce_to is set to "other". This command will prefix all announcements. This is useful for sending announcements to another channel or specifically to somebody else.

/announce_to string
The default place to send announcements. Possible values include: "echo", "emote", "other", "party", "say", "status" and "think". Of these, "status" is the only one that does not send anything to the mud. If no argument is given then the current value is displayed.

/attack_command string
The command that is used when /attack_method is set to "other". This command will be used in place of "use" or "cast". This is useful for Rangers who use Bows where the command is "fire".

/attack_method string
The method used when starting attacks with the basic: "/k", "/kk", "/kx" and "/ka". Possible values include: "use", "cast" or "other".

/attack_skill string
The skill to use when /attack_method is "use". This should be your primary attack skill.

/attack_spell string
The spell to cast when /attack_method is "cast". This should be your primary attack spell.

/beep_auto_target boolean
Beep whenever an auto target is set.

/chest_dir string
The direction (from your main chests) to move when chesting/unchesting. This is used to dump all non-kept items from your inventory so that you do not accidentally chest/equip the wrong items. This setting depends on /chest_dir_back to be set.

/chest_dir_back string
The direction (to your main chests) to move when chesting/unchesting. This should be the opposite path as /chest_dir.

/class_caster boolean
Is your incarnation int/wis/spell based? This states that your character is roughly of type "caster". The reporting of no spell points is enabled with this option.

/class_fighter boolean
Is your incarnation str/dex/con/skill based? This states that your character is roughly of type "fighter". Things like using the "kill" command to initiate combat are defined with this setting.

/commander string
The name of the current party commander. This setting is automatically configured when typing "party status". If you are not in a party this setting will be your character's name.

/echo_attr string
Attribute to use when echoing text to the status. See "/help attributes" for more details.

/effects_list string
List of effects that you care to see when typing "/cpl" or "/cpm". The format of each effect should be lower case, all spaces should be converted to underscores and all other characters should be removed. For example, something like "iron will" becomes "iron_will" and "winter's rebuke" becomes "winters_rebuke".

/email string
The default e-mail address to use when using the "/mail" command. This is useful when sending yourself notifications. Look at the "mail.cfg" in the trigs folder for more settings.

/enemies string
List of players who should automatically be targetted when entering the room, casting a spell or hitting you. This is useful so you can quickly retaliate without having to type their name.

/enemy_emotes string
The emotes to use when searching for enemies with the "/find_enemies" command. These emotes are used against your enemy in a way similar to /target_emotes. If more than one is given then a random emote will be selected each time you try to find your enemy. If an emote works then your target will be set. Possible values include: "amuse", "annoyed", "calm", "chicken", "burn", "care", "confuse", "curtsey", "duck", "grimace", "howl", "loathe", "beg", "moo", "pinch", "pity", "pretend", "sad", "whine", "why", "wtf" and "zzz".

/error_attr string
Attribute to use when echoing errors to the status. See "/help attributes" for more details.

/gac_extra string
Extra commands that should be executed with "/gac". These commands are executed after opening and getting everything from the chests, but before wearing and closing the chests. You can stick commands which wear and then re-chest your "Doomshell", or other commands to temporarily deal with items that you wish to leave in chests.

/gac_extra_post string
Extra commands that should be executed with "/gac". These commands are executed after all of your items have been pulled from the chests. You can stick commands like "bload all", or other commands that load up containers here.

/gac_extra_pre string
Extra commands that should be executed with "/gac". These commands are executed before dropping your non-kept inventory.

/give_noeq_target boolean
Should all of your non-kept inventory be given to target? This option is checked for "initiator" skills, such as "battlecry", "kiru" or "ki". This is also checked when using "/ks".

/heal_command string
The command that is used when /heal_method is set to "other". This command will be used in place of "use" or "cast". This is useful for using special items that heal you.

/heal_method string
The method used when starting heals with "/h". Possible values include: "use", "cast" or "other".

/heal_skill string
The skill to use when /heal_method is "use". This should be your primary heal skill.

/heal_spell string
The spell to cast when /heal_method is "cast". This should be your primary heal spell.

/idle_time time
The time after which triggers should stop sending data to the mud. This is useful for preventing your character from unidling when you are not present.

/ld_at_boot boolean
Should your character go ld at boot to try and get crash recoveries? This will cause your character to go link dead 40s before boot hits.

/logging string
Should world output be logged? This will cause a daily log to appear in the format of yyyy-mm-dd.log in your logs/WORLD directory.

/my_party_color string
If set this will set the "party colour" after creating a new party. Possible options include: "red", "green" or "blue".

/my_party_name string
The name of the party to create when entering the game. This setting is automatically configured when creating a party with "party create".

/on_alive string
Extra commands that should be executed after you are prayed or resurrected. This is useful for healing yourself or resetting some other settings.

/on_berry string
Extra commands that should be executed after finding or being given berries.

/on_enemy_killed string
Extra commands that should be executed after your locked target is killed. These commands are executed independent of the "/on_kill" setting and before any possible loot/corpse actions are completed.

/on_enter_game string
Extra commands that should be executed after you enter the game. If you have /idle_time set and wish to have something like your "bag of holding" summoned you should prefix the commands with "/send".

/on_enter_room string
Extra commands that should be executed after entering a room with "/run_path". This is useful for dropping corpses or setting somebody else as the party leader.

/on_fully_healed string
Extra commands that should be executed after you are fully healed. This setting is temporary and will only be executed once.

/on_fully_rested string
Extra commands that should be executed after you are fully rested. This setting is temporary and will only be executed once.

/on_kill_loose boolean
Should the R.I.P. trigger be a loose match? When enabled the locked target name is not checked and corpse triggers will be triggered on any "is DEAD, R.I.P." message.

/on_leave_game string
Extra commands that should be executed after you leave the game. These commands are executed after you have quit so nothing can be sent to the mud.

/on_leave_room string
Extra commands that should be executed before leaving room with "/run_path". This is useful for picking up items, leaving breadcrumbs or giving yourself party movement.

/on_loot string
Extra commands that should be exected after something is looted. This is useful for giving arrows, shurikens or other items to other players.

/on_loot_give_to string
The player/pet to give all loot to after looting a corpse. If set to "tank" the value is automatically filled with whomever is the tank.

/on_new_round string
Extra commands that should be exected whenever a new combat round is started. This is useful for spamming "party status" as a healer in eq.

/on_return_game string
Extra commands that should be executed after you return to the game from link death. These commands are also executed when entering the game.

/on_set_target string
Extra commands that should be executed when you lock your target with "/set_target".

/on_sizzle string
Extra commands that should be executed after you sizzle (full sps). This setting is temporary and will only be executed once.

/on_sksp string
Extra commands that should be executed after your next skill/spell completes. This setting is temporary and will only be executed once. The most common usage for this is to leave combat immediately after your skill/spell completes.

/on_start_attack string
Extra commands that should be exected when something is attacked with "/do_kill". This is useful for playing Samurai where you can type "skenjutsu", "sspirit" or "smwalk" before attacking.

/on_start_heal string
Extra commands that should be exected when something is healed with "/do_heal".

/on_tick string
Extra commands that should be executed after your next tick. The most common usage for this to re-enter combat immediately after you tick. This setting is temporary and will only be executed once.

/on_tick_warn string
Extra commands that should be executed after your next tick warning. The most common usage for this to leave combat just before you are about to tick. This setting is temporary and will only be executed once.

/on_unstunned string
Extra commands that should be exected after you are free from the stun/interrupt.

/pac_extra string
Extra commands that should be executed with "/pac". These commands are executed after dropping your non-kept inventory but before opening chests or removing your equipment.

/pac_extra_post string
Extra commands that should be executed with "/pac". These commands are executed after all chesting has been completed.

/pac_extra_pre string
Extra commands that should be executed with "/pac". These commands are executed before dropping your non-kept inventory. You can stick commands like "bunload all", or other commands that pull items out of containers here.

/prefix string
Prefix for all messages that are echoed to the status.

/prots_cooldown integer
The amount of time that party members must wait between requesting your currently active effects.

/quiet_mode boolean
Should the client act as quietly as possible? This setting disables all announcements, disables the target emotes and attempts to make your character be as quiet as possible. This will not work normally when in this mode and it's mostly useful for player killing.

/report_effects boolean
Should the status of your effects be reported to those around you? This displays the fact that some effect is comes up or has fallen.

/report_fatigue boolean
Should fatigue changes be reported to those around you? This setting will report any changes in your fatigue to "think". This is extremely useful for smaller parties where your damage matters.

/report_hps boolean
Should your hit point changes be reported to those around you? This is different than "sc party" in that it shows the actual changes and percents.

/report_kills boolean
Should the killing of your locked target be announced to those around you? This is useful for letting those not currently in the room know about combat being completed.

/report_scans boolean
Should the current shape of your locked target be reported to those around you? This is useful for letting those not currently in the room know how close the target is to being killed.

/report_sksp boolean
Should skills/spells be reported to those around you? This setting is checked during "/do_prot" and "/do_heal" and reports what you are casting/using. This option is similar to /report_target.

/report_sps boolean
Should your spell point changes be reported to those around you? This is different than "sc party" in that it shows the actual changes and percents.

/report_target boolean
Should offensive skills/spells be reported to those around you? This setting is checked during "/do_kill" and reports the skill and the target you are casting/using to "party". This is especially useful when leading a group as you want to share the exact target.

/report_ticks boolean
Should your ticks be reported to those around you? This setting will report to "think" the fact that you ticked and how many seconds it took for your tick.

/report_warnings boolean
Should general warnings be reported to those around you? This setting will report things such as incoming spells or the fact that the locked target has arrived to "party". This is useful when leading a group of vulnerable players.

/save_interval time
The time between automatically saving your settings.

/scan_target boolean
Should your target be scanned? This setting used to control the new round scan but that's been replaced with the prompt. This just controls whether or not you scan/look at the target for other things such as "/run_path" or "/set_target". This is useful for monsters that have specials that harm you when scanning them such as the Lumberjack.

/start_attack_command string
The command that is used when /start_attack_method is set to "other". This command will be used in place of "use" or "cast".

/start_attack_method string
The method used when starting attacks with the basic: "/ks". Possible values include: "use", "cast" or "other".

/start_attack_skill string
The skill to use when /start_attack_method is "use". This should be your primary start attack skill such as "battlecry", "kiru" or "ki".

/start_attack_spell string
The spell to cast when /start_attack_method is "cast". This should be your primary start attack spell such as "mental illusions".

/status_attr string
Attribute to use for your status bar. You will want it to be something that stands out from the normal mud output. See "/help attributes" for more details.

/status_pad string
The character that should be used to pad your status. This character will populate all regions of your status not filled with other data. A character such as "_" or "#" are recommended.

/tank string
The name of the current party tank. This setting is automatically configured when typing "party status". If you are not in a party this setting will be your character's name.

/target_emotes string
List of emotes that should be used when locking a target. If more than one is given then a random emote will be selected each time you lock your target. Possible values include: "wobble", "smooch", "drool", "french", "threaten", "surprise", "mercy", "disarm", "puppyeyes", "snuggle", "forbid", "applaud", "compliment", "bad" and "fullmoon". As a Devil you can also use "dchallenge" and "dgrin" which are particularly useful since they aren't messed up with targets who also happen to be verbs.

/tick_show time
The amount of time s since your last tick to show in the status bar. As long as your tick is less than this setting your status bar will be updating every second. After which your status will only be updated with events.

/tick_sps integer
The number of spell points to change before a tick is recognized. You will want to keep this setting as close to your bare minimum as possible as you will want to avoid having things that are not ticks be registered as such.

/tick_time time
The time to wait before warning you about an incoming tick. A total of 2 warnings will be issued, the second of which will be this setting x2.

/ticking boolean
Is your incarnation one which ticks? This states that you receive ticks and all things tick related should be used. If you are a Death Knight you will want to turn this off.

/title string
The title you wish your character to have. Some things like balloons cause your title to change and this helps restore your title to your desired text.

zombie/abjurer.tf

/aura_effects_list string
List of effects that you care to see when typing "/aura_effects" or "/aura_effects_missing". The format of each effect should be lower case, all spaces should be converted to underscores and all other characters should be removed. For example, something like "iron will" becomes "iron_will" and "winter's rebuke" becomes "winters_rebuke". The format is exactly the same as /effects_list.

/report_aura string
The format in which the aura should be repoted. Possible values include "list" (shows all effects in your list), "missing" (shows just the effects that are offline), "online" (shows all effects that are online) and "off". When set to "list" or "missing" it is required that you set /aura_effects_list.

/report_aura_party boolean
Should the result of aura detection automatically be reported to the party? This calls "/aura_effects_party", "/aura_effects_online_party" or "/aura_effects_missing_party" depending on the value of /report_aura. If /report_aura is off then nothing is done.

/vuln_cmd string
The name of the macro (without leading /) you wish to use as your main vuln. This is used by the "/v" command.

zombie/arrows.tf

/arrows boolean
Are you currently making arrows? When enabled it automatically removes completed arrows from the sphere and grabs /asphere when you are at 85% sps.

/quivers integer
The number of quivers that you are currently making. This setting is automatically configured when you "count quiver". When one quiver is full it will give the Nth quiver to yourself.

zombie/casts.tf

/report_casts_on_rip boolean
Should the last spell you started casting be reported when something dies? This setting will report to "think" the name and the number of rounds left before completing. This setting is useful when playing a supporting role where you wish the tank of the party to know that they should not move yet.

zombie/death_knight.tf

/give_slices_to string
The player who should receieve slices of flesh as you make them. Possible values include "me" and "tank" as well as a specific player with whom you wish to share.

/report_drains boolean
Should suffocations and flesh rots be reported to those around you? When enabled it will report the number of drains completed to the "party".

/slices_fatigue float
The amount of fatigue you regen from eating a slice of flesh. This setting is used to best determine how many you should eat in a single sitting.

/slices_heal integer
The amount of healing you can expect from eating a slice of flesh. This setting is used to best determine how many you should eat in a single sitting.

zombie/dwarf.tf

/drunk boolean
Is your character currently drunk? This set is automatically set after you've consumed a beverage; it is disabled when you sober up.

/have_alcohol boolean
Does your character currently have alcohol to drink? When you have alcohol it will automatically be consumed whenever you are sober and are in battle. This setting is automatically set when you "drink" the first time; it is disabled when your keg is empty.

/kegs integer
The number of kegs to swap between. This setting is automatically set when you "count keg".

zombie/healer.tf

/heal_all_cast time
The number of seconds it normally takes to cast "heal all". This number is used to determine when you are warned to cast "heal all" and will subtract from the cooldown.

/heal_all_wait time
The number of seconds you normally have to wait between casting "heal all".

zombie/languages.tf

/report_languages string
Should a change in your knowledge of a language be reported to those around you? Languages change from reading scrolls and from being in a room with a chatty non-player character.

zombie/logging.tf

/ignore_commands string
List of commands that you wish to not have logged. Possible values include: "whisper", "shout", "emoteto", "tell", "say", "emote", "think", "bell" or "transfer".

/log_channels string
List of channels to be logged. Channels which are contained within this list will be logged to logs/WORLD/messages.log.

/log_self boolean
Should your own messages be logged? If you only wish to see what other people say you should enable this. This has no affect on the logging of channels.

zombie/monk.tf

/house_owner boolean
Are you currently the owner of your house? If you are the house owner then others will be able to update their house plans using "add", "remove", "insert", "clear" and "help" on the "house" channel.

zombie/princess.tf

/report_princess boolean
Should the fact that the princess is probably ready to be freed be reported to those around you?

zombie/samurai.tf

/samurai_auto_scharge boolean
Should scharge automatically be executed when you use life forces from the glove and you have enough spell points? This makes the grinding of masteries much more pleasant.

/samurai_sdrain_delay float
The number of seconds to wait before executing "sdrain" on a corpse. This setting is useful to allow others to "rend" or loot the corpse first.

/samurai_sword_dam_max integer
The maximum damage of your samurai blade. This is used to calculate the percentage of your boost as well as used to prevent attempting to boost something that is already at max.

/samurai_sword_dam_min integer
The starting damage of your samurai blade. This is used to calculate the percentage of your boost.

/samurai_sword_element string
The element (damage type) that you wish your sword to maintain. The "/boost_sword" macro will try and set this element whenever it's down. Possible values include: "fire", "acid" and "poison".

/samurai_sword_hit_max integer
The maximum hit bonus of your samurai blade. This is used to calculate the percentage of your boost as well as used to prevent attempting to boost something that is already at max.

/samurai_sword_hit_min integer
The starting hit bonus of your samurai blade. This is used to calculate the percentage of your boost.

/samurai_sword_special string
The special that you wish your sword to maintain. The "/boost_sword" macro will try and set this special whenever it's down. Possible values include "jigoku" and "tenrai".

/samurai_sword_summoned boolean
Do you currently have your sword summoned? This setting is automatically configured when you type "eq", "slots" or summon your sword.

/samurai_sword_wc_max integer
The maximum weapon class of your samurai blade. This is used to calculate the percentage of your boost as well as used to prevent attempting to boost something that is already at max.

/samurai_sword_wc_min integer
The starting weapon class of your samurai blade. This is used to calculate the percentage of your boost.

/samurai_sword_wielded boolean
Do you currently have your sword wielded? This setting is automatically configured when you type "eq", "slots" or "wield" your sword.

zombie/sksp.tf

/report_sksp_changes boolean
Should percentual changes in your skills and spells be reported to those around you? The skills/spells change from equipment, bards, talismen, tales, masteries, etc. This reports how much your skills/spells changed.

zombie/trader.tf

/bag boolean
Do you currently have a "bag of holding" summoned? This setting is automatically configured when you summon your bag.

/booth_dirs string
The directions (from Central Square) to your booth. The paths should be separated by a semi-colon. This is used for "/csbooth", "/check_booth" as well as "/da" and "/wd". This setting depends on /booth_dirs_back to be set.

/booth_dirs_back string
The directions (from your booth) to Central Square. This should be the opposite path as /booth_dirs.

/stuff_bag boolean
Should your "bag of holding" be used to store loot from kills? This setting is useful for incarnations where you have a very low percent of "summon bag of holding" and you wish to only keep important items such as keys or potions in the bag.
