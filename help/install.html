Install Guide

Requirements

You need to have tf5.0b8 with the tf-python patch to use these scripts. You can get TinyFugue from http://tinyfugue.sourceforge.net and you can get the tf-python patch from http://sizer99.com/tf/. You will also need Python 2.5 or later installed on your system. Compiling tf will require access to the libraries.

On my system I installed the python2.6-dev package from Ubuntu's repository and linked to it with the following configure command:

./configure --prefix=$HOME --with-libdirs=/usr/lib/python2.6

After configuring completed I followed up with:

make
make install

Getting the Source

You need an up-to-date version of the Zombii archive which you can get with the following git command:

git clone git://github.com/schrepfer/zombii.git

Setting up TF

To setup tf run the script called install.sh.

./zombii/install.sh

Answer the questions as you are prompted. If you do not wish to save your password inside of the .tfrc go ahead and hit enter while it asks you for your password.

By default tf installs executables to your $HOME/bin directory. Make sure you have that added to your $PATH variable.

Starting

To start up tf it's as simple as typing the name of your world. By default the worlds are of the format zombie-[character]. If you chose to have the symlink created, you can simply type the name of your world:

zombie-conglomo

If you opted out of having the symlink created you can load up tf by typing:

tf

If you use the symlink option, tf is started inside of gnu screen. That will allow you to have your account always connected to the mud. All you have to do is type the name of your world and it'll restore your previous session. If a previous session does not exist a new one will be created.
