/alias [options] -- variable [value]

Gets the value of the alias from the mud and sets the local copy.

options:
-f
Force this request even if idle/away.
variable:
The variable that you wish to retrieve or update. The variable is stored on the mud as the _variable alias.
value:
Value to set the variable to. If omitted then the value is retrieved from the mud.

/chest_equipment [name] [start] [end]

Puts equipment away in chests in range. It uses %{chest_dir} and %{chest_dir_back} to organize equipment before chesting; meaning that it will attempt to drop all unkept items so that you do not accidentally chest items that are junk. This command assumes that the chests are already unlocked with /uc. This automatically closes chests after it is done. The %{chest_name}, %{chest_start} and %{chest_end} so other chest commands can be executed without specifying this again. You may also access this through /pac.

name:
The name of the chests to open. If no name is specified then the generic 'chest' will be used.
start:
The number at which to start opening chests.
end:
The number to which to end opening chests.

example:
/pac figsam 1 3

/def_auto_target [options]

Define an auto target macro which is created as auto_target_TANK_COUNT. This automatically escapes all regular expressions and makes it easier to add auto targets. It also allows you to not worry about how to match the party strings. You may also access this through /add_auto_target.

options:
-lLEFT
The text to the left of the target.
-nCOUNT
The number of auto targets tank has.
-rRIGHT
The text to the right of the target.
-tTANK
The tank announcing targets.

example:
If your tank has messages that look like Conglomo [party]: BC --> bob <-- then you can create an auto target with the following command: /def_auto_target -t'conglomo' -n0 -l'BC --> ' -r' <--'

/do_announce [options] -- message

The main trig interface to displaying messages. This macro will announce the message to the default medium unless otherwise specified. Will also only delegate to the other announce_to macros when %{announce} is true or otherwise forced. You may also access this through /say.

options:
-a
Announce regardless of the %{announce} var's setting.
-b
Do not print customized brackets (party).
-cCOLOR
Prints the message with the color given.
-dDESTINATION
The destination of the message. Without this it uses the %{announce_to} variable.
-fSTATUS
Prints a status for the message using separator.
-m
Only send messages to party IF there's more than one member (party).
-nREPEATS
Number of times to repeat the message.
-t
Send messages if and only if you are the tank (party).
-x
Never send messages to status when other channel fails.
message:
The message that is to be announced.

/do_heal [options]

The main interface for which to cast/use heals.

options:
-aACTION*
The action to perform. Must be one of 'cast', 'use' or 'other'.
-d
Do not scan the target before healing.
-nNAME
The proper name of this skill/spell.
-q
Do not announce that you are about to heal.
-sSKSP*
The skill or spell to perform.
-tTARGET*
The target to heal.
-xSPEED
Casting speed. Must be one of 'very quick', 'quick', 'normal', 'slow' or 'very slow'.

example:
/do_heal -a'cast' -s'heal body' -t'conglomo' -x'very quick'

/do_kill [options]

The main interface for which to cast/use attacks. This also triggers @on_start_attack.

options:
-aACTION*
The action to perform. Must be one of 'cast', 'use' or 'other'.
-i
Defines this as an initiator sksp and does some extra actions such as enabling %{give_noeq_target}.
-k
Kill the target as well as start skill/spell.
-m
Use kill all instead of kill when -k is enabled.
-nNAME
The proper name of the skill/spell.
-q
Do this attack as quietly as possible.
-sSKSP*
Name of the skill or spell to perform.
-tTARGET*
The target to kill.
-v
Verbose. Announce always.
-xSPEED
Casting speed. Must be one of 'very quick', 'quick', 'normal', 'slow' or 'very slow'.

example:
/do_kill -a'use' -s'ki' -k -i -t'giftah'

/do_prot [options]

The main interface for which to cast/use protective spells/skills.

options:

-aACTION*

-dDESTINATION
The destination of the message. Without this it uses the %{announce_to} variable.
-nNAME
The proper name of this skill/spell.
-q
Do not announce the performing of action.
-sSKSP*
The name of the skill/spell to perform.
-tTARGET
The target on which to perform action.
-xSPEED
Casting speed. Must be one of 'very quick', 'quick', 'normal', 'slow' or 'very slow'.

/egg [options] -- [message]

This is basic egg timer implementation. It's used to setup small reminders.

options:
-a
Announce the rate to your default %{announce_to} location.

(ignore) Specifies that this is the initial setup of the egg and enables the ability to setup the repeats.
-rREPEATS
The number of times to repeat message. Each repeat comes 30 seconds after the previous.
-s
Stop the timer.
-tDELAY
Set the delay for the timer.
-x
(ignore) This is run from a /repeat pid. It is used to distinguish between typing /egg and actually displaying the timer at the right time.

The message to display when the egg timer goes off.

example:
/egg -t30 -r2 -- Take out the trash!
